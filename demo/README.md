# DataMaster Demo 样例与验收

这里的样例用于验证 DataMaster 的多表合并、经营分析、利润分析、客户/产品分析，以及 Excel、Word 报告导出。所有金额均为虚构的人民币元数据，不含任何真实客户或密钥。

## 如何选择样例

- 快速体验单月分析：只上传 `samples/经营明细_2026-01.csv`。
- 验证多表合并和跨月趋势：一次同时选择 `samples/经营明细_2026-01.csv`、`samples/经营明细_2026-02.csv`、`samples/经营明细_2026-03.csv`。这是推荐的 Demo 路径。
- 验证经营改进建议：合并三个月后，重点查看持续亏损的“便携风扇”和亏损客户“松林电商”，再用已配置的 DeepSeek 或阿里云百炼生成建议。

三个文件采用相同的 UTF-8 表头：

```text
日期,产品,客户,数量,收入,成本,费用
```

系统按 `利润 = 收入 - 成本 - 费用` 计算。三个月合并后的基准结果如下，可用于人工核对：

| 指标 | 预期值 |
| --- | ---: |
| 明细行数（保留全部原始行） | 28 |
| 收入 | 240,900 元 |
| 成本 | 175,950 元 |
| 费用 | 49,100 元 |
| 利润 | 15,850 元 |
| 利润率 | 6.58% |
| “便携风扇”利润 | -10,850 元 |
| “松林电商”利润 | -5,500 元 |

样例有意保留三类轻量数据质量问题，但都能继续分析：

1. 2026 年 2 月有一笔负数量、负收入和负成本的退货记录。
2. 2026 年 3 月有一笔客户为空的记录，应归为“未填写”并给出质量提示。
3. 2026 年 3 月末有两条完全重复记录，应给出重复提示；Demo 默认保留原始行，便于核对。

## 启动与手工体验

1. 按项目根目录的说明启动 Spring Boot 后端。
2. 浏览器打开后端提供的 GUI 页面。
3. 在“数据分析”区域同时选择上述三个 CSV，点击“上传并分析”。
4. 在“AI 配置”区域分别保存 DeepSeek、阿里云百炼配置。API Key 只在本机使用；样例和测试脚本均不包含真实 Key。
5. 查看总览、产品、客户和月度结果，再分别导出 `.xlsx` 与 `.docx`。

未配置 API Key 时，确定性经营分析与报告导出仍应可用；只有 AI 洞察需要有效 Key。

## 自动冒烟测试

后端启动后运行：

```bash
./demo/smoke-test.sh
```

可通过环境变量修改地址和输出目录：

```bash
BASE_URL=http://127.0.0.1:8765 OUTPUT_DIR=/tmp/datamaster-smoke ./demo/smoke-test.sh
```

脚本不依赖 `jq`，不会写入或测试真实 API Key。它会验证：健康接口、多 CSV 上传、DeepSeek/百炼配置读取，以及 Excel/Word 导出。成功导出的文件默认保存在 `demo/output/`。
